# Buttons

<Subtitle>
  Buttons are UI elements which allow users to take actions throughout the app.
  It is important they are identifiable, consistent, communicate their actions clearly,
  and are appropriately sized to their action.
</Subtitle>

## Styles

Dialtone provides three levels of visual importance for buttons within three different visual styles.
The three levels of visual importance are:

1. **Clear**: This button communicates the least importance. It offers no background or border color.
It appears as a link with matching padding. This is typically used for secondary or
minimally important actions. This is the default importance level of all buttons.

2. **Outlined**: This button communicates slightly more importance than clear. It provides a border color,
but no background color.

3. **Primary**: This is the highest visual importance a button can have.
It provides a border and a background color.

These button styles are provided in three visual styles are:

1. **Default / Blue**: Our normal blue button colors.
2. **Danger**: Buttons associated with potentially destructive actions. Appears as red.
3. **Inverted**: For scenarios when you want to place a button on a non-white.

## Default

The default button offers three button styles. To help provide clarity to users, it is recommended
that only one button have the primary button style within a section or page.

> **Please note** that all button classes have hover, active, focus, and disabled states within them.
Specific classes are provided for the disabled state so this can be applied to objects that
aren't buttons (or can't use the disabled attribute).

|name        |prop value             |Default                                       |Disabled
|---         |---                    |---                                           |---
|**Default** |`importance='clear'`   |<Story id="elements-buttons--clear" />   |<Story id="elements-buttons--disabled" />
|**Outlined**|`importance='outlined'`|<Story id="elements-buttons--outlined" />|<Story id="elements-buttons--disabled" />
|**Filled**  |`importance='primary'` |<Story id="elements-buttons--default" /> |<Story id="elements-buttons--disabled" />

### Usage

```js
import { DtButton } from '@dialpad/dialtone-vue';
```

```vue
<dt-button importance="clear/outlined/primary">
  Button
</dt-button>
```

## Danger

The danger button style is used to communicate destructive actions such as deleting content, accounts, or canceling services.

```js
import { DtButton } from '@dialpad/dialtone-vue';

  Danger Button
</dt-button>
```

## Inverted

The inverted button style is used to visually separate buttons set on darker backgrounds. This can be used in
conjunction with the `importance` prop for different kinds of inverted buttons.

<Canvas>
  <Story id="elements-buttons--inverted" />
</Canvas>

### Usage

```js
import { DtButton } from '@dialpad/dialtone-vue';
```

```vue
<dt-button inverted>
  Button
</dt-button>
```

## Icons

Buttons can optionally have icons passed through the `icon` named slot. When you pass an icon, use the `iconPosition`
prop to control the position of the icon within the button (unless using a circle button).

<Canvas>
  <Story id="elements-buttons--with-icon" />
</Canvas>

### Usage

```js
import { DtButton } from '@dialpad/dialtone-vue';
```

```vue
<dt-button>
  Button
  <icon-star #icon />
</dt-button>
```

## Circle Buttons

An icon-only button in a circle. When using a circle button, the `icon` slot should be passed.

<Canvas>
  <Story id="elements-buttons--circle" />
</Canvas>

### Usage

```js
import { DtButton } from '@dialpad/dialtone-vue';
```

```vue
<dt-button circle importance="outline">
  <icon-camera #icon />
</dt-button>
```

## Link Buttons

A button that is styled as a link. Link buttons should be used whenever you have an element that is clickable and
styled as a link, but does not transition the user to a different route.

<Canvas>
  <Story id="elements-buttons--link" />
</Canvas>

### Usage

```js
import { DtButton } from '@dialpad/dialtone-vue';
```

```vue
<dt-button link>
  Link
</dt-button>
```
